# ashare-system-v2 设计目标与演进蓝图

> 本文档记录系统的长期设计目标、核心设计原则，以及量化专家视角下对现有骨架的诊断与方向定义。
> 是所有后续实现任务（见 `implementation-tasks.md`）的顶层依据。

---

## 一、系统核心定位

**不是**一个单纯的量化信号系统，**是**一个以 A 股市场为战场、以 Agent 团队为大脑、以 QMT 为执行手的**自进化智能交易体系**。

三个关键词：
- **自进化**：系统能从每次交易结果中学习，自动调整策略权重、因子权重、风控阈值
- **Agent 团队**：不是单一 AI，是多角色协作的决策委员会（选股、风控、执行、元治理）
- **A 股原生**：所有设计必须尊重 T+1、涨跌停、政策干预等 A 股特有规则

## 二、核心诊断：当前骨架的五个关键缺口

### 缺口 1：自进化闭环断裂

```
现状：交易执行 → 结果落地 → [断路] → 策略/模型维持不变
目标：交易执行 → PostMortem归因 → 因子IC评估 → 模型增量更新 → 策略权重自适应 → 再交易
```

`learning/` 模块已有 `attribution.py`、`self_evolve.py`、`trade_review.py`，但：
- `SelfEvolver` 的 `suggest_*` 方法仅输出建议，**没有写回 `RuntimeConfigManager`**
- `TradeAttributionRecord` 已有 `next_day_close_pct` 但**没有驱动因子 IC 滚动更新**
- 缺少**触发机制**：收盘后自动启动归因 → 进化循环

### 缺口 2：市场状态感知缺失

当前三大策略（动量/均值回归/放量突破）**并联运行**，无市场状态过滤层：
- 动量策略在熊市/震荡市会持续亏损
- 均值回归在单边上涨行情中错失机会
- 缺少**牛/熊/震荡/高波**状态机来动态切换主导策略

### 缺口 3：执行质量未被量化

- 无冲击成本估算（大单直接市价，实际损耗被吞没）
- 无 TWAP/分批执行逻辑
- 无滑点归因（成交价 vs 信号价的差距未被记录和学习）

### 缺口 4：因子有效性无在线监控

`FactorEngine` 能计算 150+ 因子，但：
- 无 **IC 滚动监控**（因子是否仍然有效）
- 无 **因子正交化**（同源因子导致信号集中暴露）
- 无 **自动降权/停用**机制

### 缺口 5：Walk-Forward 验证缺失

`BacktestEngine` 是事件驱动、T+1 无前视偏差，基础扎实，但：
- 无 Walk-Forward 框架（滚动扩窗训练+测试）
- 参数优化 (`optimizer.py`) 存在过拟合风险
- 无 **OOS（样本外）性能追踪**

## 四、A 股特有设计原则（不可妥协）

1. **T+1 铁律**：所有买入后，available 数量 = 0，次日才可卖出。回测和实盘均强制执行。

2. **涨跌停流动性陷阱**：
   - 买入涨停股时，必须估算封板资金和成交概率
   - 持仓股跌停时，触发尾盘最后 3 分钟等待解封逻辑
   - 连续跌停（N 板）时，按最悲观假设计入隔日清仓

3. **政策扰动优先于模型信号**：
   - 重大政策发布时（如证监会公告），`EmergencyStop` 优先于任何 AI 信号
   - RegimeDetector 应接入政策事件感知（基于公告关键词）

4. **北向资金是 A 股最强外生信号**：
   - 北向净流入持续 3 日以上 → 牛市偏多信号
   - 北向单日净流出超 50 亿 → 触发防御模式

5. **板块轮动比个股选择更重要**：
   - A 股存在明显的板块效应，同板块个股高度联动
   - 选股漏斗中，板块动量判断应先于个股因子打分

6. **A 股流动性分层极端**：
   - 日均成交 < 5000 万的股票，大单直接影响价格
   - 冲击成本模型必须考虑流动性分层

## 五、Agent 团队角色定义

| Agent | 角色 | 核心能力 | 触发时机 |
|---|---|---|---|
| `ashare-meta` | 元治理 Agent | 评估其他 Agent 表现、调整权重、触发进化 | 每周日收盘后 |
| `ashare-alpha` | 选股 Agent | 运行选股漏斗、因子打分、候选池管理 | 每日盘前 |
| `ashare-risk` | 风控裁判 Agent | 执行 6 大规则 + A 股扩展规则、情绪保护 | 每笔意图提交时 |
| `ashare-execution` | 执行优化 Agent | 冲击成本估算、执行算法选择、滑点监控 | 订单生成时 |
| `ashare-postmortem` | 复盘 Agent | 平仓归因、因子 IC 更新、学习集维护 | 每日收盘后 |
| `ashare-research` | 研究 Agent | 北向资金分析、龙虎榜解读、市场状态播报 | 按需/每日早报 |

## 六、数据分层目标

| 层级 | 数据类型 | 延迟要求 | 当前状态 | 目标状态 |
|---|---|---|---|---|
| L0 | EOD 日频数据（K线/财务） | T+1 可接受 | akshare ✓ | 保持，增加数据质量校验 |
| L1 | 日内分钟数据 | < 3 分钟 | xtquant 未启用 | 接入 QMT subscribe_quote |
| L2 | 实时 Tick / Level-2 | < 1 秒 | 完全空白 | 接入 QMT Level-2（优先级 P2） |
| L3 | 另类数据（北向/融资/龙虎榜） | T+1 可接受 | 部分 akshare | 系统化接入，纳入因子库 |

## 七、成功指标定义

系统进化的效果应能被量化验证，定义以下关键指标：

| 指标 | 计算方式 | 目标 |
|---|---|---|
| 年化收益率 | 组合净值年化 | > 20%（超越沪深300 10pct） |
| 最大回撤 | 净值峰谷比 | < 15% |
| 夏普比率 | 年化超额收益 / 波动率 | > 1.5 |
| 因子 IC_IR | 滚动 IC 均值 / IC 标准差 | 主力因子 IC_IR > 0.5 |
| 选股胜率 | 买入后 5 日正收益比例 | > 55% |
| 执行滑点 | 成交均价 vs 信号价差 | < 0.3% |
| 自进化频率 | 每周至少 1 次参数自适应 | 100% 执行 |
