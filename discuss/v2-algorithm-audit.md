# ashare-system-v2 算法诊断与优化方案

> 逐模块审查 v1 现有算法，识别数据污染风险和策略缺陷，给出 v2 优化方向。

## 一、v1 算法问题总诊断

### 严重程度分级

| 等级 | 含义 |
|------|------|
| **致命** | 会直接导致错误交易决策 |
| **严重** | 严重降低策略有效性 |
| **中等** | 影响系统可靠性或准确度 |
| **轻微** | 可优化，但不影响正确性 |

## 二、逐模块诊断

### 2.1 数据层 — research_service.py

#### 致命: 假数据注入 pipeline

```python
# research_service.py:108-110
if not requests:
    mock = MockResearchAdapter()
    return mock.sync_news(symbol)  # ← 真实数据拉不到就注入假利好！
```

**问题:** 当 AkShare 接口失败时，系统 fallback 到 Mock 数据，注入一条"季度利润大幅超预期"的假利好新闻。这意味着：
- 数据源异常时，系统反而会收到虚假的正面信号
- 评分引擎把假利好当成真信号，可能触发错误买入
- **线上运行时 AkShare 接口不稳定是常态**，这个 fallback 会频繁触发

**v2 修复:**
- 数据获取失败 → 返回空列表，而非假数据
- 标记数据来源和质量等级 (real / cached / unavailable)
- 下游评分时，数据不可用 ≠ 利好，应该降分或跳过

---

#### 严重: NLP 情感分析是关键词计数

```python
# research_service.py:63-71
positive_keywords = ["超预期", "增长", "利好", "重组", "增持", "提升", "突破", "中标"]
negative_keywords = ["净亏", "下跌", "风险", "减持", "跌停", "下滑", "处罚", "回落"]
pos_count = sum(1 for kw in positive_keywords if kw in text)
neg_count = sum(1 for kw in negative_keywords if kw in text)
```

**问题:**
- 仅 8 个正面 + 8 个负面关键词，覆盖面极窄
- 无法处理否定句："并非利好" → 被判为 positive
- 无法处理程度："微幅增长" vs "爆发式增长" 得分相同
- "重组"未必是利好 (也可能是被迫重组/破产重整)

**v2 修复:**
- Phase 1: 扩展关键词词典到 200+，按强度分级 (强/中/弱)
- Phase 2: 接入 NLP 情感模型 (FinBERT 或 LLM 打分)
- 增加否定句检测 ("并非"/"不是"/"未能" 反转情感)

### 2.2 评分引擎 — scoring.py

#### 严重: 简单加法模型，无标准化

```python
# scoring.py:115
total = 50.0 + event_score + tech_score + sector_score + leader_score
        + liquidity_score + position_score - spread_penalty
```

**问题:**
- 基准分 50 意味着什么都不知道的股票已经有 50 分 → 系统天然偏向"买入"
- 各维度量纲不统一: event_score [-30, 30], tech_score [-30, 30], sector_score [-15, 15], leader_score [-10, 12]
- 板块得分最大 ±15，但事件得分最大 ±30 — 权重分配不合理且无实证依据
- 加法模型无法捕捉因子交互 (比如"板块强+资金流入"的协同效应)

**v2 修复:**
- 所有因子先标准化到 [0, 1] 区间
- 使用加权几何平均或 XGBoost 替代简单加法
- 权重通过回测历史数据优化，而非拍脑袋
- 基准分改为 0，不引入偏置

---

#### 严重: 买卖决策过于粗糙

```python
# scoring.py:148-149
if score >= strong_buy_threshold and pos_events > neg_events:
    return "BUY"
```

**问题:**
- 只有 BUY/SELL/HOLD 三个状态，缺少信号强度
- "正面事件数 > 负面事件数" 是数量比较，一条重大利空可能比三条小利好更重要
- 没有考虑市场环境 (情绪周期)
- 阈值 (70/60/80) 是硬编码，未经回测验证

**v2 修复:**
- 输出连续信号强度 (0-100)，而非离散 BUY/SELL
- 买入信号 = 评分 × 环境系数 × 情绪系数
- 阈值通过回测自动寻优

---

### 2.3 技术指标 — indicators.py

#### 中等: 指标覆盖面不足

**现状:** 仅 4 个指标 (MA, MACD, RSI, 量比)

**缺失的关键指标:**

| 指标 | 用途 | 重要性 |
|------|------|--------|
| ATR (真实波动幅度) | 止损距离计算、仓位管理 | **必须有** |
| 布林带 (Bollinger) | 波动率通道、超买超卖 | 高 |
| KDJ | 短线超买超卖 (A股常用) | 高 |
| OBV (能量潮) | 量价背离检测 | 中 |
| VWAP | 盘中均价基准 | 做T必须 |
| 换手率分位数 | 相对于历史的换手活跃度 | 高 |
| 资金流入/流出 | 主力资金方向 | 高 |

#### 中等: RSI 算法不精确

```python
# indicators.py:90-100 (Cutler's RSI, 非 Wilder's)
avg_gain = sum(gains) / period
avg_loss = sum(losses) / period
```

**问题:** 使用简单平均而非指数加权平均 (Wilder's RSI)，与市面上所有行情软件的 RSI 不一致，导致信号偏差。

**v2 修复:** 改用 Wilder's 指数平滑 RSI。

### 2.4 选股漏斗 — screener.py

#### 严重: 宏观过滤形同虚设

```python
# screener.py:75-76
if sh_price > 0:  # ← 只要上证指数有报价就认为"市场正常"
    return ScreenerResult(...)
```

**问题:** 这一层名义上是"极端行情过滤"，但实际上只检查"价格 > 0"，永远不会拦截任何行情。

**v2 修复:**
- 涨停/跌停家数比 (跌停 > 涨停 × 3 → 极端弱势)
- 大盘近 N 日涨跌幅 (< -3% → 系统性风险)
- 两市成交额 (< 5000亿 → 流动性不足)
- 情绪周期阶段 (冰点 → 仅允许低吸，禁止追高)

---

#### 中等: 板块热度采样噪声大

```python
# screener.py:134-137
sample = members[:10]  # ← 每个板块只取前10只
# ...
avg_change = sum(changes) / len(changes)  # 仅看1天涨跌幅
```

**问题:**
- 每板块仅采样前 10 只，可能偏向大盘股，遗漏真正的热点个股
- 仅看 1 天涨跌幅，噪声极大，可能选到"当天反弹但整体下跌"的板块

**v2 修复:**
- 板块热度 = 3日涨幅 × 0.5 + 5日涨幅 × 0.3 + 资金净流入 × 0.2
- 采样扩大到 20-30 只，或用板块指数直接计算
- 加入连续性判断 (连涨 ≥ 2 天的板块优先)

### 2.5 风控 — risk.py

#### 中等: 仓位计算过于粗糙

```python
# runtime_engine.py:295-296
target_buy_value = min(balance.cash * 0.2, (balance.total_asset or balance.cash) * 0.1)
```

**问题:**
- 固定比例 (现金的 20%，总资产的 10%)，不考虑标的波动率
- 高波动股票和低波动股票分配相同资金 → 高波动股亏损杀伤力更大

**v2 修复:**
- **ATR 仓位法:** 仓位 = 账户风险额度 / (ATR × 倍数)
- **凯利公式:** f* = (p × b - q) / b，其中 p=胜率, b=赔率, q=败率
- **情绪系数:** 情绪阶段决定仓位上限 (冰点 20%, 回暖 60%, 主升 80%, 高潮 30%)

---

#### 中等: 止损止盈是固定比例

```python
# runtime_engine.py:285-286
stop_loss=round(snapshot.last_price * 0.95, 3)    # 固定 -5%
take_profit=round(snapshot.last_price * 1.08, 3)   # 固定 +8%
```

**问题:**
- 所有股票统一 5% 止损、8% 止盈
- 不考虑个股波动率: 高波动股 5% 可能是正常日内波动 → 频繁止损
- 盈亏比固定 1:1.6，没有根据市场环境调整

**v2 修复:**
- 止损 = 入场价 - N × ATR (波动率自适应)
- 止盈 = 入场价 + M × ATR (M > N 确保正期望)
- 移动止损: 盈利后逐步上移止损位
- 分批止盈: 到达第一目标位减半仓，剩余仓位用移动止损

---

### 2.6 板块/龙头评估 — theme_engine.py / leader_selector.py

#### 中等: 龙头识别过于简单

```python
# leader_selector.py:21-25
if change_pct >= 3.0:
    momentum_component = 8.0
elif change_pct >= 1.0:
    momentum_component = 4.0
```

**问题:** 仅用当日涨幅 + 成交量判断"龙头"，真正的龙头识别应考虑：
- 板块内相对强度 (涨幅排名前 3)
- 首板/连板情况
- 是否率先封板
- 跟风 vs 领涨的区分

**v2 修复:**
- 龙头评分 = 板块内涨幅排名 × 0.3 + 封板时间 × 0.2 + 量能 × 0.2 + 连板数 × 0.3
- 接入涨停/连板数据

## 三、数据质量治理方案

### 3.1 数据污染源识别

| 污染源 | 当前状态 | 影响 |
|--------|----------|------|
| Mock 假数据 fallback | 数据拉取失败→注入假利好 | **致命: 可能触发错误买入** |
| 停牌股混入 | 未过滤 | 报价为 0，导致计算异常 |
| 除权除息未处理 | 未处理 | MA/MACD 等指标在除权日产生假信号 |
| ST/退市整理股 | 仅名称匹配 | 可能遗漏 (代码不含 ST 字样) |
| 次新股数据不足 | 仅检查上市天数 | K线不足时技术指标不可靠 |
| 集合竞价期间报价 | 未区分 | 09:15-09:25 报价不稳定 |

### 3.2 数据清洗 pipeline (v2 新建)

```
原始数据 → [阶段 1: 过滤] → [阶段 2: 修复] → [阶段 3: 标准化] → 干净数据
```

**阶段 1: 过滤 (剔除不可用数据)**
- 停牌股 (成交量 = 0 且涨跌幅 = 0)
- 上市不足 20 天的次新股
- ST / *ST / 退市整理期 (查询 xtdata 名称字段)
- K 线不足 60 根的标的 (技术指标不可靠)

**阶段 2: 修复 (处理数据缺陷)**
- 除权除息: 使用前复权价格计算技术指标
- 缺失值: 前值填充 (ffill)，超过连续 3 个缺失则标记为不可用
- 异常值: 涨跌幅超过 ±11% 的非涨跌停日 (可能是数据错误)

**阶段 3: 标准化 (因子标准化)**
- 截尾 (Winsorize): 去掉 1% / 99% 分位的极值
- Z-Score 标准化: 因子值 → 标准正态分布
- 行业中性化: 去除行业效应，只保留个股 alpha

## 四、策略体系升级方案

### 4.1 评分模型升级路线

```
v1 (当前):  加法模型 (硬编码权重)
    ↓
v2-Phase1:  加权评分 + 因子标准化 (权重通过回测优化)
    ↓
v2-Phase2:  XGBoost 模型 (自动学习因子权重和交互)
    ↓
v2-Phase3:  多模型集成 (XGBoost + Transformer + 规则)
```

### 4.2 选股漏斗升级

```
v1:  宏观(形同虚设) → 板块(1天采样) → 技术(4指标)
     ↓
v2:  环境评分(情绪+宏观+资金) → 板块轮动(多日+资金流) → 因子筛选(500+因子)
     → AI精选(XGBoost评分) → 风控放行(凯利仓位+情绪保护)
```

### 4.3 仓位管理升级

```
v1:  固定 20% 现金分配
     ↓
v2:  凯利公式基础仓位 × ATR 波动调整 × 情绪周期系数 × 相关性惩罚

     具体公式:
     base_position = kelly_fraction × account_equity
     volatility_adj = target_risk / (ATR_14 / price)
     emotion_coeff = sentiment_phase_ceiling
     correlation_adj = 1 - portfolio_correlation_penalty

     final_position = min(base_position, volatility_adj) × emotion_coeff × correlation_adj
```

### 4.4 止盈止损升级

```
v1:  固定 -5% 止损, +8% 止盈
     ↓
v2:  三层止损机制

     1. 初始止损 = 入场价 - 2 × ATR_14
        (波动率自适应，高波动股止损更宽)

     2. 盈利保护止损 (移动止损):
        当浮盈 > 1 × ATR 时，止损上移至入场价 (保本)
        当浮盈 > 2 × ATR 时，止损上移至入场价 + 1 × ATR
        持续跟随，最大只回撤 1.5 × ATR

     3. 时间止损:
        持仓 T+3 后仍无盈利 → 触发卖出评估
        (避免资金长期被套在横盘股)

     止盈:
     - 第一目标: 入场价 + 3 × ATR → 减仓 50%
     - 第二目标: 入场价 + 5 × ATR → 减仓 30%
     - 剩余 20%: 移动止损跟踪
```

## 六、优化建议总结 (按紧急度排序)

### 立即修复 (不修就会出错)

1. **删除 Mock 数据 fallback** — 数据拉取失败返回空列表，不注入假数据
2. **宏观过滤层真正生效** — 加入涨跌停比、大盘涨跌幅、成交额检查
3. **停牌股过滤** — 在数据获取后立即过滤成交量为 0 的标的

### 尽快完成 (显著提升策略有效性)

4. **评分模型标准化** — 所有因子归一化到 [0,1]，去掉 50 分基准偏置
5. **ATR 计算 + 波动率自适应止损** — 替代固定 5%/8%
6. **凯利仓位管理** — 替代固定 20% 资金分配
7. **情绪周期系统** — 影响仓位上限，避免高潮期满仓
8. **数据清洗 pipeline** — 除权处理、缺失值、异常值

### 中期建设 (从规则到 AI)

9. **因子引擎 + IC 验证** — 500+ 因子，但只用有效的
10. **XGBoost 替代加法评分** — 自动学习因子权重
11. **回测引擎** — 用数据验证策略，而非拍脑袋

### 长期进化

12. **NLP 模型替代关键词** — 情感分析质量飞跃
13. **Transformer/LSTM** — 序列预测
14. **自进化闭环** — reviewer 自动优化参数
