## 一、数据层设计

### 数据源分类

```
原始数据
├── 行情数据
│   ├── Level1：分钟/日K（OHLCV）
│   ├── Level2：逐笔成交、委托队列、盘口快照
│   └── 指数：上证、深证、创业板、北证、行业指数
├── 基本面数据
│   ├── 财务报表（季报/年报）
│   ├── 分析师预期（一致预期EPS/营收）
│   └── 公司事件（增减持、回购、解禁、股权激励）
├── 另类数据
│   ├── 新闻舆情（NLP情感分数）
│   ├── 北向资金（陆股通实时持仓变化）
│   ├── 融资融券（余额、净买入）
│   └── 龙虎榜（机构/游资席位）
└── 衍生数据
    ├── 涨停板专项（封板时间、封单量、炸板次数）
    ├── 板块联动（题材涨停数、扩散速度）
    └── 市场情绪（情绪指数、赚钱效应）
```

### 数据清洗方案

**A股特有脏数据问题及处理：**

1. **停牌/复牌跳空**：标记停牌区间，复牌首日不参与截面排名
2. **ST价格限制差异**：区分±5%和±10%/±20%，涨停识别用不复权价格
3. **除权除息不连续**：前复权用于技术指标，后复权用于收益率，不复权用于涨停识别
4. **新股/次新股失真**：上市不足60日不参与选股
5. **Level2时间戳对齐**：区分集合竞价、连续竞价、尾盘竞价三个session
6. **极值处理**：用MAD（中位数绝对偏差）替代3σ，更适合A股肥尾分布

## 二、因子体系

### 有效因子分类

| 类别 | 代表因子 | 适用周期 |
|------|---------|---------|
| 价量因子 | 动量、反转、换手率、量价背离、涨停因子 | 短线 |
| 板块联动因子 | 题材热度、板块内排名、资金轮动、连板梯队 | 短线 |
| 市场微观结构 | 大单净买入、委托失衡、价格冲击、Amihud流动性 | 日内 |
| 基本面因子 | ROE、成长性、估值分位、资产负债 | 中长线 |
| 另类因子 | 北向资金、融资情绪、舆情、龙虎榜 | 短中线 |

### 因子有效性检验标准
- IC_IR > 0.5 认为因子有效
- 需检验：IC均值、IC_IR、分位数收益差、因子换手率、IC衰减曲线、分市场状态稳定性

## 三、多Agent协商决策框架

### Agent职责划分

```
Orchestrator（协调者）
├── MarketStateAgent    市场状态识别（强趋势/轮动/防守/混沌）
├── SectorLinkageAgent  板块联动分析（breadth/扩散/轮动）
├── StockSelectionAgent 个股排序（按战法选择排序逻辑）
├── RiskControlAgent    风险控制（一票否决权）
└── ExecutionMonitorAgent 实时退出监控
```

### 协商机制
- 不是投票，而是职责分离 + 否决权
- 状态机决定激活哪些Agent
- 风控Agent有一票否决权
- 其他Agent只有推荐权

### 战法路由

| 市场状态 | 激活战法 |
|---------|---------|
| 强趋势 | 龙头追板、分歧回封 |
| 题材轮动 | 板块回流首板、低位补涨 |
| 退潮防守 | 仅低吸核心，或空仓 |
| 混沌震荡 | 空仓 |

## 四、退出逻辑（战法化）

| 场景 | 触发条件 | 动作 |
|------|---------|------|
| 开仓失败立撤 | 买入后5分钟内未按预期方向运动 | 无条件平仓 |
| 冲高不封减仓 | 冲击涨停但未封住，或封板后炸板 | 立即减仓50-100% |
| 板块退潮清仓 | 所属题材涨停数下降，板块强票走弱 | 不管盈亏清仓 |
| 时间止损 | 超过战法预设持仓时间窗口 | 收盘前强制平仓 |

---

## 五、回测注意事项（A股特有）

1. **T+1制度**：当日买入不能当日卖出
2. **涨跌停无法成交**：涨停买不进，跌停卖不出
3. **冲击成本**：参与率超过20%时冲击成本显著
4. **未来函数**：财报用披露日，分析师预期用发布时间戳
5. **幸存者偏差**：必须包含历史退市股票

## 六、量化优势发挥点

| 环节 | 量化优势 |
|------|---------|
| 市场状态识别 | 实时计算全市场指标，不遗漏 |
| 候选筛选 | 全市场扫描，速度快 |
| 股性画像 | 统计历史行为，客观 |
| 退出执行 | 纪律性强，不拖延 |
| 风控 | 仓位计算、集中度控制 |

---

## 七、核心设计原则

1. **宁可错过，不可做错** — 状态不明时空仓
2. **退出优先于买入** — 系统80%精力在"什么时候走"
3. **战法纯粹性** — 每个Agent只做一件事
4. **可解释性** — 每笔交易必须能追溯到哪个Agent、哪个信号
5. **数据质量 > 模型复杂度** — 脏数据上跑再复杂的模型都是垃圾进垃圾出

---

## 八、实现优先级

```
第一步：市场状态机（最高ROI，能空仓就是最大风控）
第二步：退出规则战法化（把固定止损改成场景化退出）
第三步：板块内相对强弱（替换绝对涨幅排名）
第四步：个股股性画像（60日滚动统计，每日更新）
第五步：多Agent协商框架（前四步是它的输入）
```
