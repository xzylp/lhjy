# QMT 接口使用说明

## 1. 目的

这份文档说明当前项目已经实现并验证可用的 Windows 侧 QMT 对外接口，包括：

- 行情查询
- 账户资产查询
- 持仓查询
- 当日委托查询
- 当日成交查询
- 下单
- 单笔委托状态查询

约定方式是：

- Linux 或其他调用方只访问 Windows 对外接口 `18791`
- Windows 内部自动转发到项目内管理员态交易桥
- QMT 程序和运行数据固定使用当前项目目录内的脚本、配置、状态文件

## 2. 当前运行结构

### 2.1 对外接口

- 对外统一入口：`http://<Windows主机IP>:18791`
- 本机测试入口：`http://127.0.0.1:18791`

### 2.2 内部交易桥

- 内部管理员态交易桥：`http://127.0.0.1:18792`

说明：

- `18792` 仅供 Windows 本机代理转发使用
- Linux 不需要直接访问 `18792`
- Linux 只需要访问 `18791`

### 2.3 鉴权

- Header 名称：`X-Ashare-Token`
- token 文件：`.ashare_state/ops_proxy_token.txt`

当前项目内固定位置：

- Token 文件：[ops_proxy_token.txt](C:\Users\yxzzh\Desktop\XTQ\.ashare_state\ops_proxy_token.txt)
- 主配置文件：[windows_gateway_config.json](C:\Users\yxzzh\Desktop\XTQ\config\windows_gateway_config.json)

## 3. 调用总规则

### 3.1 请求头

除 `/health` 外，其他接口都需要带：

```http
X-Ashare-Token: <token内容>
```

### 3.2 返回格式

所有接口统一返回 JSON。

通用判断方式：

- `ok = true` 表示接口执行成功
- `ok = false` 表示接口执行失败
- 失败时看 `last_error`、`message`、`diagnostics`

### 3.3 价格与交易方向

下单接口当前支持：

- `side=buy`
- `side=sell`

当前支持的价格类型：

- `fix`
- `fixed`
- `limit`
- `latest`
- `market`

其中：

- `fix/fixed/limit` 映射为限价
- `latest/market` 映射为最新价

### 4.2 状态总览

#### 请求

`GET /status`

#### 作用

返回以下状态：

- 对外代理进程信息
- QMT 进程状态
- 内部管理员交易桥状态
- gateway worker 状态
- 日志路径

#### 示例

```bash
curl -H "X-Ashare-Token: <token>" http://127.0.0.1:18791/status
```

#### 关键字段

- `proxy.is_admin`
- `qmt.running`
- `qmt.pid`
- `trade_bridge.reachable`
- `trade_bridge.port`

只要出现：

- `qmt.running = true`
- `trade_bridge.reachable = true`

就表示交易链路已经可以工作。

### 4.9 下单

#### 请求

`POST /qmt/trade/order`

#### 请求体

```json
{
  "stock_code": "600000.SH",
  "side": "buy",
  "quantity": 100,
  "price": 9.9,
  "price_type": "fix",
  "strategy_name": "windows_gateway",
  "remark": "linux_order_001"
}
```

#### 字段说明

- `stock_code`：证券代码
- `side`：`buy` 或 `sell`
- `quantity`：委托数量
- `price`：委托价格
- `price_type`：`fix` 或 `latest`
- `strategy_name`：策略名
- `remark`：备注

#### 示例

```bash
curl -X POST \
  -H "X-Ashare-Token: <token>" \
  -H "Content-Type: application/json" \
  -d "{\"stock_code\":\"600000.SH\",\"side\":\"buy\",\"quantity\":100,\"price\":9.9,\"price_type\":\"fix\",\"strategy_name\":\"windows_gateway\",\"remark\":\"linux_order_001\"}" \
  http://127.0.0.1:18791/qmt/trade/order
```

#### 返回结构

```json
{
  "ok": true,
  "submitted": true,
  "intent_id": "",
  "last_error": "",
  "connect_attempts": [
    {
      "session_id": 123456,
      "connect_result": 0
    }
  ],
  "session_id": 123456,
  "connect_result": 0,
  "subscribe_result": 0,
  "order_id": 1082134344
}
```

#### 成功判断

下单成功需要同时满足：

- `ok = true`
- `submitted = true`
- `connect_result = 0`
- `subscribe_result = 0`
- `order_id > 0`

## 5. 委托状态码说明

当前最常用状态码：

- `48`：未报
- `49`：待报
- `50`：已报
- `51`：已报待撤
- `52`：部成待撤
- `53`：部撤
- `54`：已撤
- `55`：部成
- `56`：已成
- `57`：废单

当前项目里已经实际查到过：

- `50`：已报
- `54`：已撤

## 7. Python 调用示例

```python
import json
import urllib.request

BASE = "http://127.0.0.1:18791"
TOKEN = open(r"C:\Users\yxzzh\Desktop\XTQ\.ashare_state\ops_proxy_token.txt", "r", encoding="utf-8").read().strip()

def api_get(path):
    req = urllib.request.Request(
        BASE + path,
        method="GET",
        headers={"X-Ashare-Token": TOKEN},
    )
    with urllib.request.urlopen(req, timeout=60) as resp:
        return json.loads(resp.read().decode("utf-8"))

def api_post(path, payload):
    req = urllib.request.Request(
        BASE + path,
        method="POST",
        headers={
            "X-Ashare-Token": TOKEN,
            "Content-Type": "application/json; charset=utf-8",
        },
        data=json.dumps(payload, ensure_ascii=False).encode("utf-8"),
    )
    with urllib.request.urlopen(req, timeout=60) as resp:
        return json.loads(resp.read().decode("utf-8"))

asset = api_get("/qmt/account/asset")
print(asset)

order = api_post(
    "/qmt/trade/order",
    {
        "stock_code": "600000.SH",
        "side": "buy",
        "quantity": 100,
        "price": 9.9,
        "price_type": "fix",
        "strategy_name": "windows_gateway",
        "remark": "python_test",
    },
)
print(order)
```

## 9. 注意事项

### 9.1 休市下单

如果在非交易时段下单：

- 委托可能进入下一交易日队列
- 也可能被券商或柜台直接撤掉

因此状态需要以 `order_status` 和 `trades` 的实际回查为准。

### 9.2 管理员权限

这套实现的关键点是：

- QMT 安装在系统盘
- 交易连接需要管理员权限
- 外部调用不需要管理员权限
- 由项目内管理员交易桥负责完成实际交易连接

### 9.3 Linux 侧前提

Linux 要真正发单，需要满足：

- 能访问 Windows 的 `18791` 端口
- 知道 token
- 按本文档的 JSON 结构发送请求
