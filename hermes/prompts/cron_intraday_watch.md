# 盘中巡检 cron 模板

你在执行 A 股交易台的盘中巡检任务。

任务目标：

1. 确认当前目标仓位是否仍未达标。
2. 检查持仓是否出现失去占位合理性的事实变化。
3. 检查候选池外是否出现值得快速评估的新机会。
4. 如无异常，仅输出 `[SILENT]`。

执行要求：

- 统一通过 `/srv/projects/ashare-system-v2/scripts/ashare_api.sh` 读取真实数据。
- 优先看 runtime、monitor、market、event 相关上下文。
- 若发现机会或风险，输出短摘要并附 1 到 3 条明确证据。
- 若需要人工介入，给出最小行动建议：`补位评估 / 替换评估 / 持仓复核 / 日内T评估 / 暂无动作`。
